# Geographical Expansion Plan: Contributor Base

**Project:** WanderAI — AI Travel Itinerary Planner
**Author:** Aryan Agarwal
**Status:** Hackathon submission — forward-looking growth plan

## How I plan to expand the contributor base geographically

WanderAI's contributor growth follows expanding circles outward — starting
where trust and access are already highest, then widening as the project
proves it can absorb outside contributions cleanly.

**Phase 1 — code.swecha.org community (Telangana):**
Start with the platform WanderAI is already hosted on. SWECHA's own student
and developer community is the most natural first contributor base — easy to
reach, already familiar with the codebase's hosting platform, and well
positioned to test and improve the existing Telugu localization specifically.

**Phase 2 — Indian college FOSS clubs & hackathon networks:**
Expand outreach to FOSS clubs and open-source cells at other colleges (e.g.
FOSS United chapters, college coding communities) across India. This is also
the natural group to recruit for **community-driven translation** of the
Hindi interface and, later, additional Indic languages (Tamil, Bengali,
Marathi) called out in the feature roadmap — each new language ideally gets
at least one native-speaking contributor to review AI-generated translations
for accuracy.

**Phase 3 — Wider open-source community (GitHub, global):**
Once the project has a track record of merged external PRs, list it on
open-source contribution aggregators (e.g. "good first issue" boards) and
push visibility through the GitHub mirror. This opens contribution to a
global pool for core engineering tasks (PDF export, map integration, test
coverage) that don't require India-specific travel knowledge.

**Phase 4 — Region-specific contributors for adjacent markets:**
As the product roadmap extends toward Bangladesh (BDT), Sri Lanka (LKR), and
Nepal (NPR) pricing support, recruit contributors based in or familiar with
those countries specifically — both to validate currency/travel assumptions
in the AI prompt and, eventually, to contribute Bengali, Sinhala, or Nepali
localization, mirroring how the Hindi/Telugu support was built for India.
